# 小范围盐丘 FWI 模拟与照明归一化差异说明

## 结论

本次小范围 FWI 模拟用于展示速度模型反演闭环：通过真实盐丘裁剪模型合成观测记录，再从平滑初始模型出发迭代降低数据残差。它和已有 RTM 照明归一化的目标不同，二者应作为互补结果解释。

## RTM 中的照明归一化

现有 RTM 代码中的震源照明归一化为 `I / (L_s + epsilon)`，源检双照明归一化为 `I / (sqrt(L_s L_r) + epsilon)`。二者作用在偏移成像结果上，主要用于削弱照明强弱造成的成像幅值差异。

在已有盐丘 RTM 结果中，震源照明和接收照明来自正传波场、反传波场能量的累计。归一化后的图像仍然是偏移剖面，不是速度模型反演结果。

## 论文中的照明优化

论文中的照明优化通常不只是图像除以照明场，而是把照明用于采集评价、双向传播覆盖分析、梯度预条件、Hessian 近似或分步多参数反演。它更接近反演过程中的条件改善，而不是单纯的成像显示增强。

因此，不能把 RTM 里的 `I / L_s` 或 `I / sqrt(L_s L_r)` 直接等同为论文中的完整照明优化 FWI。前者主要改变成像幅值，后者通常影响反演更新方向、更新尺度或观测系统评价。

## 与已有盐丘 RTM 结果的关系

已有 full RTM 诊断中低照明区占比约为 `0.0174`，说明当前 full aperture 盐丘 RTM 结果不是主要受照明缺失控制。源检双照明归一化与震源照明归一化保持较高相似性，Laplacian 和显示优化对结构可读性的影响更明显。

这意味着现有 RTM 结果更适合表述为“照明条件整体足够，图像解释质量主要受显示、低波数噪声和成像条件影响”。如果继续讨论照明优化，应把重点转向 FWI 梯度预条件或采集照明评价，而不是只比较偏移图像除以不同照明场。

## 本次 FWI 演示的意义

FWI 直接比较预测记录和观测记录，通过数据残差更新速度模型。它回答的是“当前速度模型是否能解释观测数据”，而 RTM 照明归一化回答的是“已有偏移图像的幅值是否被照明不均衡影响”。

本次小范围演示采用局部盐丘模型、少量炮点和较短时间步，优先得到可复现的误差下降曲线、速度模型对比和炮记录对比。该结果可以作为论文或报告中的方法验证图，说明从 RTM 成像进一步过渡到 FWI 速度更新的基本思路。

## 后续扩展方向

后续若需要更贴近论文中的照明预处理 FWI，可在本脚本基础上增加照明预条件梯度缩放，对比未预条件和预条件两组误差曲线、速度更新分布和炮记录拟合效果。

## 照明预条件 FWI 对比

新增的照明预条件对比实验保留原始 baseline FWI，同时增加 `illumination_preconditioned` 更新模式。该模式先累计正传震源波场能量，得到近似震源照明场：

`L_s(x,z) = sum_t u_s(t,x,z)^2`

然后将残差反传得到的更新方向 `g(x,z)` 按归一化照明进行缩放：

`g_pre(x,z) = g(x,z) / (L_s_norm(x,z) + epsilon)`

这里的 `g_pre` 是速度模型更新方向，不是 RTM 偏移图像。因此，该对比实验的判读重点是两组数据残差曲线、速度更新分布和最终炮记录拟合，而不是偏移图像亮度是否更均衡。

这也对应论文方法和当前 RTM 代码之间的关键差异：RTM 照明归一化作用在成像结果 `I` 上，主要改善显示和振幅均衡；FWI 照明预条件作用在反演梯度或更新方向 `g` 上，主要改善模型更新尺度和迭代稳定性。

## 照明预条件参数扫描

照明预条件中的 `epsilon` 控制低照明区域的更新放大程度。`epsilon` 越小，低照明区域的更新越容易被放大；`epsilon` 越大，预条件越保守。为了避免只用单一参数得出片面结论，新增参数扫描模式：

```powershell
python -m admit_fwi.run_small_salt_fwi --scan-illumination --iterations 3 --nt 360
```

默认扫描：

`epsilon = 0.01, 0.02, 0.05, 0.10, 0.20`

扫描结果应按数据残差下降比例判读：

- 如果某个 `epsilon` 的下降比例超过 baseline，说明该参数下照明预条件有助于当前小范围反演。
- 如果所有 `epsilon` 都弱于 baseline，说明当前轻量预条件形式没有直接带来误差优势，需要进一步联合调整步长、平滑半径或采用更接近 Hessian 的预条件。
- 无论哪种结果，都不能把该小范围参数扫描直接外推为完整盐丘多炮 FWI 结论。

## epsilon 与 max_update 二维扫描

一维扫描只改变 `epsilon`，但照明预条件会整体改变更新方向的尺度，因此原来的 `max_update` 未必适合预条件后的梯度。二维扫描同时改变 `epsilon` 和单次最大速度更新量：

```powershell
python -m admit_fwi.run_small_salt_fwi --scan-illumination-2d --iterations 3 --nt 360
```

默认扫描：

- `epsilon = 0.05, 0.10, 0.20, 0.50`
- `max_update = 20, 35, 50, 80`

二维扫描的判读重点是热力图中哪一组参数获得最大误差下降比例。如果最优组合仍低于 baseline，说明当前轻量照明预条件形式并不适合直接作为主结果；如果某一组合超过 baseline，则可以进一步围绕该组合做更细参数扫描和更大窗口验证。

## 自适应步长线搜索

二维扫描中如果不同 `max_update` 得到完全相同或高度接近的结果，说明更新方向在幅值裁剪前已经较小，单纯增大最大更新量不会改变模型。此时更合理的改进是对每次迭代的更新方向做步长线搜索：

```powershell
python -m admit_fwi.run_small_salt_fwi --line-search --iterations 3 --nt 360
```

默认测试：

`step_scale = 0.25, 0.5, 1.0, 1.5, 2.0`

每次迭代会对所有候选步长分别计算预测记录误差，选择误差最低的候选模型。如果所有候选都不能降低误差，则保留当前模型。该实验可以判断固定步长是否限制了 baseline 或照明预条件 FWI 的收敛。

线搜索结果的判读方式：

- baseline 线搜索明显优于固定步长，说明固定更新尺度限制了反演。
- 照明预条件线搜索优于 baseline，说明照明预条件方向需要配套步长选择。
- 照明预条件线搜索仍弱于 baseline，说明当前轻量照明预条件方向本身仍不占优，需要改进梯度构造或采用更严格 Hessian 近似。

## 自适应扩展线搜索

如果线搜索每轮都选择候选列表中的最大步长，说明当前搜索上界可能偏低。自适应扩展线搜索先测试较小候选：

`step_scale = 0.5, 1.0, 2.0`

如果最优步长落在当前最大值且误差继续下降，则继续测试：

`step_scale = 3.0, 4.0, 6.0, 8.0`

运行命令：

```powershell
python -m admit_fwi.run_small_salt_fwi --adaptive-line-search --iterations 3 --nt 360
```

该实验用于判断前一阶段的 `step_scale=2.0` 是否只是人为上界。如果 baseline 在扩展后继续明显改善，说明 FWI 更新主要受步长上界控制；如果照明预条件扩展后仍弱于 baseline，则更能说明当前轻量照明预条件方向本身不占优。
